# 前端纠偏计划：回到竞赛原始产品叙事

## 背景

当前 `investment-team` 页面在连续迭代后，前端重心逐渐收缩到：

- 对话
- 画板
- SOP 流程编辑

这与最初竞赛设计文档中的产品主叙事已经出现偏离。

原始设计文档的核心不是“一个 SOP 工作台”，而是一个完整的“医药生物投资决策智能体系统”，核心应围绕以下四条一级能力展开：

1. Copilot 分析入口
2. Multi-Agent 团队协作
3. 风险雷达 / 推演引擎
4. 员工画像 / Skill / 协作闭环

## 当前偏离点

1. `SOP` 被做成了主入口，甚至接近主产品形态，但它在原始设计中更接近员工经验沉淀层。
2. `风险推演` 在当前页面中几乎不可见，弱化了竞赛方案的重要差异化亮点。
3. `团队协作` 被缩成成员展示，而不是“发起项目、拉人进组、协同分析、沉淀结论”的完整协作视角。
4. `对话` 和 `画板` 成了产品一级导航，但原始设计里它们更应该隶属于 `Copilot 分析` 能力。

## 纠偏目标

只调整前端信息架构和页面表达，不修改：

- 后端接口
- Prisma schema
- Agent 逻辑
- 工作流执行逻辑

通过前端重组，把当前已有能力重新挂回原始竞赛叙事中。

## 新的信息架构

`investment-team` 页面调整为四个一级主视图：

1. `Copilot 分析`
   - 主入口
   - 承载“分析面板 + 分析画板”
   - 对话和画板在这里统一呈现

2. `团队协作`
   - 项目负责人
   - 团队成员
   - 协同任务 / 最近 session
   - 多角色协作说明

3. `风险推演`
   - 风险雷达
   - 情景推演卡片
   - 影响评估展示
   - 作为竞赛亮点恢复到一级层级

4. `员工画像 / SOP`
   - 员工个人画像
   - Skill / SOP 资产
   - 当前可复用分析流程
   - SOP 不再作为全页面主角，而回到“经验沉淀”位置

## 页面组织原则

1. 顶层先讲产品，不先讲编辑器。
2. `对话` 与 `画板` 是 `Copilot 分析` 的内部工作区，不再与团队、风险、员工画像同级竞争。
3. `SOP` 是个人能力资产，不再作为产品默认入口。
4. 页面视觉优先服务比赛展示逻辑，而不是内部运维式工作台逻辑。

## 实施策略

### 第一步：保留已有可复用能力

前端继续复用：

- `ChatInput`
- `ChatMessage`
- `WelcomeDashboard`
- `InfiniteCanvas`
- `useChatStream`
- `useEmployeeInvestmentWorkbench`

### 第二步：只重构 `investment-team/page.tsx`

本轮只改：

- `src/app/(dashboard)/investment-team/page.tsx`

不修改其他组件，尽量通过页面层组合完成叙事纠偏。

### 第三步：用静态前端表达补齐缺失叙事

因为本轮不改后端，所以以下内容以展示型前端表达为主：

- 风险推演区
- 通知闭环区
- 团队协作摘要
- 员工画像摘要

## 预期结果

重构后页面需要满足：

1. 用户一眼能看出这是“医药投资决策智能体系统”，而不是“一个工作流编辑器”。
2. 四条竞赛主线都能在前端中找到对应入口。
3. 现有对话、画板、团队、SOP 能力仍然存在，但层级被重新摆正。
4. 页面适合比赛展示和截图，不再继续朝内部配置台方向偏移。

## 本轮范围外

以下内容不在本轮处理：

- 新增后端 API
- 调整数据库结构
- 修改 Agent 输出格式
- 真正接通风险推演后端逻辑
- 新建项目 / 成员邀请的真实后端链路
