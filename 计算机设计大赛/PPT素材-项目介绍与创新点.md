# PPT 素材：PharmaInvest Agent — 医药生物投资决策智能体系统

---

## 一、项目概述（1页PPT）

**项目名称：** PharmaInvest Agent — 智能体赋能的医药生物投资决策支持系统

**一句话定位：** 用多智能体协作技术，为医药投资领域打造一个"AI分析师团队"，实现从数据到决策的全链路智能化。

**解决的核心问题：**
- 医药投资涉及财报、研报、政策、临床试验等多源异构数据，传统人工处理效率低、门槛高
- 现有工具存在明显短板：同花顺有 Agent 但不懂医药，Wind 有数据但没有 Agent，药融云有医药数据但不做投资决策
- **我们填补的空白：** "医药领域专业 Agent + 投资决策闭环" 这个交叉点，目前没有产品占据

---

## 二、产品三大核心功能（3页PPT，每个功能1页）

### 功能一：智能副驾（Copilot）
- **用户体验：** 用自然语言描述需求，如"分析恒瑞医药的投资价值"
- **系统响应：** 4个专业 Agent 自动并行工作（财报分析、研报解读、政策监控、竞品追踪）
- **输出结果：** 一份带数据溯源的深度分析报告，每条结论标注来源（财报第X页/研报第X页）
- **亮点：** 支持多轮对话追问，如"它和复星医药比谁更好？"→ 自动组织对比分析
- **PPT建议：** 可用对话界面截图 + 报告输出示例展示

### 功能二：Multi-Agent 协作团队
- **核心概念：** 不是单个AI，而是一个"分析师团队"——4个专业智能体各司其职、协同工作
  - 🔍 **财报分析 Agent** — 解读财务数据，计算关键指标
  - 📄 **研报解读 Agent** — 提取券商研报核心观点和评级
  - 📋 **政策监控 Agent** — 追踪集采、医保谈判等政策变化
  - 🏢 **竞品追踪 Agent** — 监控同赛道企业的管线进展和获批动态
- **编排层：** LangGraph Orchestrator 智能调度，决定调用哪些 Agent、以什么顺序、如何汇总
- **PPT建议：** 用流程图展示 Agent 协作过程，强调"可视化思考链路"

### 功能三：风险雷达 + 推演引擎（核心创新点）
- **风险雷达：** 实时监控药企风险信号，主动推送预警
- **推演引擎（独创）：** 不只分析现状，还能**模拟未来场景**
  - 例如：如果某核心产品纳入集采，对企业营收影响多大？
  - 基于历史真实案例（如"XX药业2022年集采后营收下降12%"）做参数化推演
  - 输出影响估算 + 置信区间 + 参考案例 + 缓冲因素分析
- **类比：** 相当于一个"医药投资沙盘"，帮助投资者在做决策前先推演风险
- **PPT建议：** 用推演示例的输入/输出对比图，最具冲击力

## 三、创新点详解（重点页，2-3页PPT）

### 创新点 1：Multi-Agent 协作架构
- **创新之处：** 不是大模型简单问答，而是多个专业智能体分工协作，模拟真实投研团队的工作模式
- **技术实现：** 基于 LangGraph 的有向图状态机编排，支持 Agent 间的动态任务分配和结果汇总
- **对比优势：** 同花顺的 Agent 是独立的，我们的 Agent 能协同推理、交叉验证

### 创新点 2：场景推演引擎（最大差异化）
- **创新之处：** 首次将"what-if 场景推演"引入医药投资分析。不是预测股价，而是**推演政策/竞品/临床事件对企业基本面的影响**
- **技术方法：** "参数化场景模板 + 历史案例检索 + LLM 推理"三段式
  - 第一步：预定义场景模板（集采纳入、竞品获批、临床失败等）
  - 第二步：RAG 检索历史同类事件的真实影响数据作为 ground truth
  - 第三步：LLM 基于历史案例 + 当前企业数据推理，给出带置信区间的估算
- **验证方式：** 用已知历史事件（如恒瑞医药2021年集采影响）做回测验证
- **为什么这很重要：** 解决了"AI 分析只看过去不看未来"的痛点

### 创新点 3：全链路数据溯源
- **创新之处：** 每条分析结论都标注数据来源（财报页码、研报段落），做到"言之有据、结论可查"
- **技术实现：** RAG 向量化时保留元数据（公司代码、年份、章节名、页码），生成时回填引用
- **为什么这很重要：** 金融领域 AI 最大的信任障碍就是"幻觉"，溯源机制直接解决这个问题

### 创新点 4：自主学习迭代
- **创新之处：** Agent 能根据用户反馈持续优化分析策略
- **技术实现：** 用户点赞/点踩 → 存入偏好数据库 → 下次分析时检索历史反馈 → 注入 Agent 提示词
- **效果：** 系统越用越"懂"用户的分析偏好和关注重点

## 四、技术架构（1页PPT）

```
┌─────────────────────────────────────┐
│         用户界面（Web 前端）          │  ← 自然语言交互 + 可视化
├─────────────────────────────────────┤
│    Copilot 编排层（LangGraph）       │  ← 智能任务分配与调度
├────────┬────────┬────────┬─────────┤
│财报分析 │研报解读 │政策监控 │竞品追踪  │  ← 4个专业 Agent
│ Agent  │ Agent  │ Agent  │ Agent   │
├────────┴────────┴────────┴─────────┤
│         RAG 知识库层                 │  ← 向量检索 + 结构化查询
│  财报库 │ 研报库 │ 宏观库 │ 药审库   │
├─────────────────────────────────────┤
│      风险雷达 + 推演引擎              │  ← 核心创新模块
│  风险监控 │ 场景推演 │ 主动预警       │
└─────────────────────────────────────┘
```

**关键技术栈：**
- Agent 框架：LangChain / LangGraph
- LLM：GPT-4 / Claude
- 向量数据库：用于 RAG 检索
- 可观测性：LangSmith 追踪全链路调用
